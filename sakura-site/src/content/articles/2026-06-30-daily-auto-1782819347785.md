## 二、全平台流媒体解锁的核心理念：DNS分流 + 路由策略

### 2.1 DNS分流的正确姿势

很多小白直接使用公共DNS（如8.8.8.8），这会导致以下问题：
- 访问Netflix时，DNS解析返回的是CDN的任意IP，而非你节点所在地区的IP。
- 访问国内网站时，DNS解析走了国外，延迟暴增。

**正确做法**：

```json
{
  "dns": {
    "servers": [
      {
        "tag": "dns-国内",
        "address": "https://dns.alidns.com/dns-query",
        "detour": "direct"
      },
      {
        "tag": "dns-国外",
        "address": "https://dns.google/dns-query",
        "detour": "proxy"
      }
    ],
    "rules": [
      {
        "domain_suffix": [".netflix.com", ".disney.com", ".hbo.com"],
        "server": "dns-国外"
      },
      {
        "domain_suffix": [".baidu.com", ".qq.com"],
        "server": "dns-国内"
      }
    ],
    "strategy": "prefer_ipv4"
  }
}
```

**注意**：`detour: "proxy"` 表示这个DNS请求会走代理节点，确保解析到节点的原生IP。而 `detour: "direct"` 则直连国内DNS。

### 2.2 路由策略：让流量“该去哪去哪”

Sing-box 的路由规则决定了哪些流量走代理、哪些直连。对于流媒体解锁，你需要：

- **流媒体域名** → 走代理（且必须走“解锁节点”）
- **国内网站** → 直连
- **广告/追踪域名** → 拒绝

**推荐配置**：

```json
{
  "route": {
    "rules": [
      {
        "domain_suffix": [".netflix.com", ".disney.com", ".hbo.com", ".spotify.com"],
        "outbound": "解锁节点组"
      },
      {
        "domain_suffix": [".baidu.com", ".qq.com", ".taobao.com"],
        "outbound": "direct"
      },
      {
        "domain_keyword": ["ads", "tracking"],
        "outbound": "block"
      }
    ]
  }
}
```

**关键点**：不要把“解锁节点”和普通节点混在一起。建议单独创建一个“流媒体节点组”，只包含经过验证的原生解锁线路。如果你还没有这样的节点，可以先去 [晚高峰稳定专线推荐](/recommend) 挑选一条。

## 四、深度解析：Sing-box 流媒体解锁的“黑科技”

### 4.1 TLS 指纹伪装：绕过平台风控

流媒体平台会检查TLS握手时的Client Hello包特征。如果指纹与真实浏览器不符，会被判定为代理流量。Sing-box支持 `tls_fingerprint` 参数，可模拟Chrome、Firefox等浏览器。

**配置示例**：
```json
{
  "outbounds": [
    {
      "type": "vless",
      "server": "your-server.com",
      "tls": {
        "enabled": true,
        "fingerprint": "chrome"
      }
    }
  ]
}
```

**选择建议**：
- `chrome`：兼容性最好，适合大多数平台
- `firefox`：对Disney+更友好
- `random`：随机指纹，但可能触发风控

### 4.2 DNS Fragment：绕过域名审查

某些地区的ISP会对DNS查询进行干扰（如DNS劫持）。Sing-box的 `dns_fragment` 功能可以将DNS查询拆分成多个片段，绕过干扰。

**配置**：
```json
{
  "dns": {
    "fragment": {
      "enabled": true,
      "size": 16
    }
  }
}
```

**注意**：开启后可能增加延迟，建议仅在遇到DNS劫持时启用。

### 4.3 域名嗅探：自动识别未匹配的流量

Sing-box支持 `sniff` 功能，可以自动识别HTTPS流量中的域名，即使你没有在规则中明确列出。

**配置**：
```json
{
  "inbounds": [
    {
      "type": "tun",
      "sniff": true
    }
  ]
}
```

这样，即使你漏掉了某个流媒体子域名，Sing-box也能自动将其路由到代理。

## 五、常见问题与排错指南

### 5.1 “解锁成功”但播放卡顿

**原因**：节点带宽不足或延迟过高。
**解决**：使用 `ping` 和 `traceroute` 测试节点延迟。如果延迟>200ms，建议更换节点。另外，检查Sing-box的日志，看看是否有大量DNS超时。

### 5.2 只有部分流媒体能解锁

**原因**：不同平台对IP要求不同。例如，Netflix要求IP必须属于家庭宽带，而Disney+对数据中心IP更宽容。
**解决**：为每个平台单独配置节点。例如，Netflix走A节点，Disney+走B节点。在Sing-box中，可以用 `domain_suffix` 区分。

### 5.3 国内网站访问变慢

**原因**：DNS分流规则不完善，导致国内域名走了国外DNS。
**解决**：检查 `dns.rules` 中的国内域名列表是否完整。建议使用 `geosite:cn` 规则（需要下载GeoIP数据库）。
