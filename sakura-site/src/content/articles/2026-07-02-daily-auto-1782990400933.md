对于重度用户（比如需要同时看流媒体、用 AI、打游戏），单节点显然不够用。Clash Verge Rev 支持“策略组”功能，你可以将多个节点（包括原生 IP 节点和普通节点）组合成一个组，实现自动切换或负载均衡。

在规则集编辑器中，将 `DOMAIN-SUFFIX,netflix.com` 的代理策略指向“流媒体解锁”策略组，将 `DOMAIN-SUFFIX,openai.com` 指向“AI 服务”策略组。这样，当你打开 Netflix 时，客户端会自动选择最快的原生 IP 节点；而打开 ChatGPT 时，又会切换到另一个节点（如果原节点被封，会自动切换）。这种“智能分流”极大提升了用户体验。

## 五、常见问题与排错指南

### 5.1 明明买了原生 IP，为什么还是解锁失败？

- **问题1：DNS 泄漏**。如果你使用了公共 DNS（如 8.8.8.8），可能会导致 IP 信息暴露。解决方案：在 Rev 中开启“DNS 策略”为“redir-host”模式，并勾选“劫持系统 DNS”。
- **问题2：节点被滥用**。原生 IP 节点如果被太多用户同时使用，IP 信用评分会下降。建议选择支持“独享原生 IP”的机场（价格较高，但稳定）。
- **问题3：客户端版本过旧**。2026 年，旧版 Clash 核心（如 1.x 系列）已不支持最新的 Mux 协议。务必升级到 Clash Verge Rev 的最新版。

### 5.2 如何验证节点是否是真正的原生 IP？

使用 `ipinfo.io` 或 `whatismyipaddress.com` 查询节点 IP 的归属地。如果显示的是“数据中心”（如 DigitalOcean、AWS），则不是原生 IP。真正的原生 IP 会显示为“ISP”或“Residential”。你还可以在 [高性价比机场精选大厅](/recommend) 中查看用户评价，很多资深玩家会分享节点的实测结果。
